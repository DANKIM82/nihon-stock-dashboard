# 日 · NIHON — Japan Stock Dashboard

> 일본 증시 밸류에이션 리서치 터미널 · Bilingual (한국어 / English) · Live data updated daily by GitHub Actions

**Live demo**: <https://dankim82.github.io/nihon-stock-dashboard/>

A single-page, bilingual valuation dashboard for 133 TSE-listed Japanese
companies. Data is fetched from Yahoo Finance every weekday after the TSE
close and committed back to the repo as `data.json` by GitHub Actions.

## 🏗 Architecture

```
┌────────────────────────────────────────────────────────────┐
│   Browser (viewer)                                         │
│   https://dankim82.github.io/nihon-stock-dashboard/        │
│          │                                                 │
│          │  fetch('./data.json')                           │
│          ▼                                                 │
│   index.html  ←── React + Recharts (CDN, no build step)    │
└────────────────────────────────────────────────────────────┘
         ▲ served by
         │
┌────────────────────────────────────────────────────────────┐
│   GitHub Pages (main branch, repo root)                    │
│   index.html · data.json                                   │
└────────────────────────────────────────────────────────────┘
         ▲ pushed by
         │
┌────────────────────────────────────────────────────────────┐
│   GitHub Actions — .github/workflows/update-data.yml       │
│   Runs Mon–Fri @ 07:30 UTC (16:30 JST)                     │
│      1. pip install -r scripts/requirements.txt            │
│      2. python scripts/fetch_data.py                       │
│      3. git commit -m "auto-update" && git push            │
└────────────────────────────────────────────────────────────┘
         │ reads from
         ▼
┌────────────────────────────────────────────────────────────┐
│   Yahoo Finance (via yfinance)                             │
│   133 tickers × .info + 1mo history                        │
└────────────────────────────────────────────────────────────┘
```

**Why this is simple**: no build pipeline, no server, no API keys, no cost.
Everything runs from a single repo on GitHub Pages.

## 🚀 First-time deployment (5 minutes)

### Step 1. Create a new repo on GitHub

Go to <https://github.com/new>. Name it **`nihon-stock-dashboard`**.
Leave it **Public** (required for free GitHub Pages; Private works if you
have Pro). **Don't** add README / .gitignore / license — we already have
them.

### Step 2. Push this folder

```bash
cd nihon-stock-dashboard
git init
git add -A
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/DANKIM82/nihon-stock-dashboard.git
git push -u origin main
```

### Step 3. Enable GitHub Pages

1. Open your repo → **Settings** → **Pages** (left sidebar)
2. **Source** = *Deploy from a branch*
3. **Branch** = `main`, **Folder** = `/ (root)`
4. Click **Save**

After ~30 seconds, your site is live at:
<https://dankim82.github.io/nihon-stock-dashboard/>

### Step 4. Give Actions permission to push

1. Repo → **Settings** → **Actions** → **General**
2. Scroll to **Workflow permissions**
3. Select **Read and write permissions**
4. Click **Save**

(Without this, the cron job can't commit `data.json` back to the repo.)

### Step 5. Run the workflow once (test)

1. Repo → **Actions** tab
2. Click **"Update stock data"** in the left sidebar
3. Click **"Run workflow"** → **Run workflow** (green button)
4. Watch it run (2–4 minutes)

When it finishes, refresh your Pages URL — you'll see the live Yahoo Finance
data replace the bootstrap sample values. After this first run, Actions will
continue automatically every weekday at 16:30 JST.

## 🧪 Running locally

### Just view the site

```bash
# Python's built-in server works great:
cd nihon-stock-dashboard
python3 -m http.server 8000
# open http://localhost:8000
```

(You need a local server because `fetch('./data.json')` doesn't work from
`file://` due to browser CORS.)

### Test the data pipeline locally

```bash
cd nihon-stock-dashboard
pip install -r scripts/requirements.txt
python scripts/fetch_data.py

# Faster subset while developing:
python scripts/fetch_data.py --limit 10 --sleep 0.1

# Then reload http://localhost:8000
```

## 🛠 Customising

### Add / remove tickers

Edit `scripts/tickers.json`. Each entry is:

```json
{
  "ticker": "6178",
  "nameEn": "Japan Post Holdings",
  "nameKo": "일본우정",
  "nameJp": "日本郵政",
  "segment": "Prime",
  "sector": "Industrials"
}
```

Tickers must be 4-digit TSE codes (yfinance will add `.T` automatically).
Valid `sector` values are the 11 in `lib/types.ts`/the HTML's sector label
map. Valid `segment` values: `Prime`, `Standard`, `Growth`.

### Change schedule

Edit `.github/workflows/update-data.yml`. The `cron` line uses UTC:

```yaml
- cron: "30 7 * * 1-5"   # 07:30 UTC = 16:30 JST, Mon–Fri
```

For daily (including weekends) use `"30 7 * * *"`. For a second run on
Sunday evening JST (to catch US Friday close movements), add:

```yaml
- cron: "30 7 * * 1-5"
- cron: "0 12 * * 0"     # Sun 12:00 UTC = Mon 21:00 JST
```

### Tune rate-limiting

If Yahoo starts rate-limiting (you'll see many `None` returns), edit the
workflow step:

```yaml
run: python scripts/fetch_data.py --sleep 0.5  # slower, safer
```

## ⚠️ Known quirks

- **yfinance is unofficial**. Yahoo can change their endpoints without
  notice. If fetches start failing, `pip install -U yfinance`.
- **Weekends / holidays**: the cron skips Sat/Sun by default. If you want
  to run after Japanese holidays, trigger manually from Actions.
- **dividendYield unit**: yfinance sometimes returns `0.0325` and sometimes
  `3.25` for the same field. The script auto-detects and normalises to
  percent.
- **debtToEquity**: yfinance returns this as a **percent** (`105.4` means a
  1.054 ratio). The script divides by 100.
- **`52WeekChange` can be null** for newly listed names. The UI renders
  them as `—`.
